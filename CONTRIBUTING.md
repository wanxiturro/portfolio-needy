# 🤝 CONTRIBUTING

## Branch Strategy
main ← production
↑
develop ← PR target
↑
feature/* | fix/* | docs/* | style/*

text

## ⚠️ Rules

- ❌ No direct pushes to `main` or `develop`
- ✅ One PR per feature/fix
- 🔀 PR to `develop`, never to `main`

## 📝 Commit Convention

| Type | Emoji | Example |
|------|-------|---------|
| feat | ✨ | `✨ ADD: random GIF system` |
| fix | 🐛 | `🐛 FIX: mobile blocker` |
| style | 🎨 | `🎨 STYLE: neon colors` |
| docs | 📝 | `📝 DOCS: update README` |
| easter | 🎮 | `🎮 EASTER: konami code` |

## 🔄 PR Process

1. `git checkout -b fix/your-change`
2. Make changes
3. `git commit -m "✨ ADD: description"`
4. `git push origin fix/your-change`
5. Open PR to `develop`
